# AI-amnTV 开发约束

## 目标

维护一条本地优先、关卡驱动、可恢复的 AI 竖屏漫剧生产流水线。默认交付规格为
1080×1920、60–120 秒、15–25 卡、含字幕与 AIGC 标识。

## 不可破坏的规则

- 文件系统是唯一事实源；不要把关键状态只留在 Agent 对话或内存。
- Studio 的 SQLite 只用于任务队列，不得把剧本、分镜、关卡或成本迁移成数据库独占状态。
- Agent 只做创作与判断，不负责状态迁移、文件写入、重试或付费调用编排。
- 付费视频调用必须经过 `videoReadiness`。
- 供应商返回任务 ID 后必须立即持久化；孤儿视频任务不得自动重提。
- 锁定剧本的场号和台词 ID 不得删除或重排；批准分镜的既有卡号同理。
- 角色、场景、服装、音色必须先锁定，镜头只引用资产 ID。
- 原始候选和 take 不覆盖；局部重做通过递增 round 保存。
- 供应商未返回价格时记为未知，不得编造成本。
- 剪映 native draft 是实验性输出，`timeline.json` 是稳定事实源。
- 已批准剧本/分镜的内容编辑必须精准撤销相关下游状态；音频与时长回填不得触发误失效。
- Studio API 默认只监听回环地址；不要暴露 `.studio`、密钥文件或项目目录中的隐藏文件。
- 导入必须先预检、提交时再次校验，并通过同文件系统暂存目录原子落盘；绝不覆盖同名系列。
- 小说或大纲导入只能保存来源草稿，不得自动排队 Agent 或触发付费供应商。
- 配音是角色表演资产，不是成片附属品；新增声音能力时必须保留合法授权、候选试听、角色声音
  圣经、上下文表演参数和人工试听关卡。
- 分镜数据模型扩展时必须记录镜头意图，并检查信息顺序、视线、轴线、反应镜头和连续性；不能
  只以镜头数量和时长判断分镜质量。
- 人物连续性必须覆盖面部、身体、服装、姿态、声音和价值观；任何一项锁定资产变化都要精准
  撤销相关下游状态。

## 修改后的最低验证

```bash
npm run typecheck
npm test
npm run build
npm run verify
```

媒体合成或 provider 接口有变化时必须运行完整 `npm run check`。测试不得调用真实云供应商。
