# 拍板记录（Decision Log）

> 记录已由用户明确拍板的方向级决策。后续设计/实现不得偏离；要改先重新拍板。

## D1 · 产品定位（2026-07-24）

**B：自用生产工具。** 目标是真实产出可发布的 AI 漫剧（anime 风格竖屏短剧）。
验收标准 = 端到端出片质量 + 单集成本可控。CLI + 本地文件即可，无多用户/计费。
保留缺口方向 C（女频专用工作流产品化）的接口：女频判断层先作为本工具内的 agent 节点做出来，用真实出片验证有效性。

## D2 · 编排形态（2026-07-24）

**B：关卡式半自动（审核门驱动）。** 流水线自动跑，固定关卡停下等人工；关卡之间全自动（含失败降级重试）。SDK 的对话能力免费附带——任何关卡都可以多聊几轮再放行，但默认路径按关卡制设计。

## D3 · 角色固定 → 双层资产库 + 四关卡（2026-07-24）

用户明确要求**人物角色固定**。落法：

- **系列级资产**（一次锁定，跨集复用）：角色定妆照（三视图/表情/服装变体）、**角色音色**、主场景图、美术风格基准。锁定后只能按资产 ID 引用，禁止镜头里临时描述人物。
- **单集级产物**：剧本、镜头表、关键帧、视频片段，全部挂在系列资产之下。

关卡（四个，不多不少）：

> ①剧本/分集确认（每集）→ ⓪定妆锁定（每系列一次，新角色增量）→ ②分镜+关键帧选图（每集）→ ③成片确认（每集）

硬约束：关键帧/视频生成工具入参强制 `character_asset_id`，未锁定资产的角色被 hook 拦截——一致性靠门禁不靠自觉。

## D4 · 四项配置（2026-07-24，「按推荐」整体拍板）

1. **输入源**：原创剧本为默认（AI 辅助从大纲生成，版权干净）；自有/公版小说改编做成可选输入分支。不碰无授权网文。
2. **生成模型**：本地动漫模型出图（M5 Pro 48GB，候选关键帧零成本无限抽）+ 云端图生视频（Vidu/可灵/即梦，供应商立项后小评测再锁定）+ MiniMax TTS 起步（本地 CosyVoice 做免费备胎）。全部藏在工具抽象后面，换供应商 = 换工具实现。
3. **出片规格**：竖屏 9:16、每集 1–2 分钟、15–25 镜头，成片带字幕 + AIGC 标识。
4. **技术栈**：TypeScript 版 Claude Agent SDK。确定性编排用普通 TS 代码，agent 只出现在需要判断的环节。
