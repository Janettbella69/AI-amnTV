# AI-amnTV 对标 LibTV

核对日期：2026-07-24。

本对标以公开的 [libtv-labs/libtv-skills](https://github.com/libtv-labs/libtv-skills) 为可验证基线。
该仓库展示了创建/查询会话、切换画布项目、上传素材和下载结果的 OpenAPI 桥接能力；复杂创作由
LibTV 后端 Agent 完成。它不等同于 LibTV 完整产品的全部内部能力。

## 产品定位

两者不是简单替代关系：

- **LibTV** 更像由远端 Agent 驱动的通用创作画布：用户给目标，服务端负责理解和生成。
- **AI-amnTV** 是漫剧生产操作系统：结构化剧本、角色/音色/场景锁定、分镜、逐卡生成、人工关卡、
  成本、QC 与交付都可审计。

因此第一阶段不把本地生产线改造成黑盒 Agent，而是把 LibTV 接成一个受控、可追踪的外部创作台。

## 已落地

### 1. 受控发送

用户必须在 Studio 中逐个选择参考素材并点击确认。后端再次验证素材位于当前系列目录内、真实格式
受支持、数量不超过 8、单文件不超过 200MB。不会把整个仓库、剧本目录或密钥交给远端 Agent。

### 2. 持久会话

会话记录保存在：

```text
episodes/<EP-ID>/external/libtv/
└── sessions/
    ├── <LOCAL-SESSION-ID>.yaml
    └── <LOCAL-SESSION-ID>/results/
```

记录包括本地/远端 ID、指令、参考素材、消息、增量序号、结果来源、错误和更新时间。若提交阶段
进程中断，系统不会根据一个模糊状态自动再次产生费用。

### 3. 安全结果回收

结果只允许来自精确的 LibTV 结果主机，禁止 HTTP、凭证、非标准端口和相似域名；下载限制为
500MB 和图片/视频类型。写入使用新文件名，不覆盖已存在的结果。

### 4. 同会话续写

每次追问都先写入带状态的本地 turn，再使用既有远端 `sessionId` 提交；参考素材仍需本次明确选择。
若调用中断，该 turn 标记为 `orphaned`，系统不根据不确定状态自动补发。消息、序号和结果继续追加
到原会话账本，不会创建一个看似相同的新会话。

### 5. 结果晋升与下游失效

已回收的图片可显式加入指定 cut 的首帧或尾帧候选。系统会转为标准 PNG、写入来源元数据并使用
不覆盖的候选编号。若该 cut 已经圈图、生成视频或合成，默认拒绝覆盖；只有用户勾选允许撤销时，
才创建新的 retake round 并精准清除该卡下游状态。

### 6. 同量表对比

“评测”页会收集 LibTV 回收结果、本地已选关键帧、已选视频 take 和最终成片。图片按人物一致性、
构图、镜头语言、干净度评分；视频另加动作与配音表演。报告同时记录分辨率、时长、帧率、音轨、
来源 provider/model 和已知成本。供应商没有返回费用时继续显示“未知”。

### 7. 可验证 dry-run

`AMNTV_DRY_RUN=1` 时，“画布”页会走同一套本地会话和结果展示流程，但只生成一张本地验收图：
不读取 `LIBTV_ACCESS_KEY`、不访问网络、不产生费用。

## 仍有差距与下一步

1. **任务可观测性。** 增加供应商耗时、失败类型和实际返回费用字段；未知费用继续显示为未知。
2. **画布编排。** 如果真实用户需要，再做“素材 → 指令 → 结果 → 选中版本”的节点视图，而不是
   先复制一个通用无限画布。
3. **实时小额验收。** 有隔离账号后验证上传路径、返回消息结构、结果 MIME 和画布 URL；验证前
   不声称生产接口已完全可用。
4. **跨供应商实测。** 使用同一创作 brief、同一参考素材和真实费用数据，对 LibTV、即梦/可灵
   provider 与本地管线做重复样本对比；当前 dry-run 只验证流程，不代表供应商质量结论。

## 明确不复制

- 不采用“任何一句话都可能触发上传/下载”的宽泛工具触发词。
- 不把任意 URL 下载到本地，也不允许自定义 API 主机绕过域名限制。
- 不让远端 Agent 修改剧本锁定、资产锁定、镜头状态或最终审核。
- 不在无法判断远端是否已接单时自动重试一次可能计费的提交。
- 不用估算价格伪装真实成本；供应商未返回费用时保持“未知”。
