# 视听表演质量规范

状态：**目标规范，部分能力尚未实现。**

本文件把剧本、人物、配音、分镜、镜头语言、画面、声音和剪辑视为同一套表演系统。实现状态
必须以代码、测试与 Studio 实际界面为准，不能因为写入本规范就宣称功能已经完成。

## 质量定义

AI-amnTV 的目标不是“生成了一段能播放的视频”，而是交付一部具有统一导演意图的剧：

- 剧本决定观众理解什么
- 人物决定观众在乎什么
- 分镜决定信息以什么顺序出现
- 镜头语言决定观众看哪里、何时产生压力
- 配音决定人物是否像一个正在思考的人
- 画面与动作提供可信度
- 音乐、环境声和剪辑控制情绪节奏

任何一项被通用模板替代，都会形成可感知的“AI 流水线感”。

## 当前基础与缺口

| 层级 | 已实现 | 下一步缺口 |
|---|---|---|
| 剧本 | 结构化场景/台词、情绪契约、关卡锁定 | 场景目标、潜台词、可表演性检查 |
| 人物 | 角色/服装/场景/音色资产锁定 | 声音圣经、姿态与语言人格、成长连续性 |
| 分镜 | 结构化卡、卡号锁定、时长回填 | 镜头意图、轴线/视线/反应镜头检查 |
| 配音 | Audio-first、逐句 TTS、实际时长回填 | 声音选角、上下文合成、表演参数、声音 QC |
| 画面 | 参考图、候选、圈图、局部重做 | 表情/姿态/服装/光线的跨镜连续性 |
| 声音后期 | 字幕、音轨和最终 QC | 空间声、环境声、响度、音乐避让和混音审核 |
| 发行 | MP4、SRT、封面、稳定时间线 | 发现版、连续观看版和角色运营包 |
| 评测 | 三检查点结构证据、人工看片/试听、不可变报告、版本过期提示、产物同量表对比 | 感知模型辅助、跨集趋势和真实供应商重复样本 |

## 当前评测闭环

Studio 的“评测”页已经实现一套不伪造感知质量的基础闭环：

- **剧本 / 分镜、样片 / 作监、成片 / 交付**三个检查点按阶段采用不同维度与权重。
- 自动检查只读取可验证的剧本、资产、分镜、镜头状态、音频/视频和交付 QC 证据。
- 每项证据标记为 `direct`、`proxy` 或 `missing`；代理证据只计一半覆盖率。
- 画面审美、人物表演、镜头意图、配音质感和连续性可由人看片/试听后评分并写下具体证据。
- 自动与人工同时存在时，维度分数按自动 45%、人工 55% 合成；覆盖不足时报告保持“待人工评审”。
- 报告保存输入哈希。上游内容或状态变化后旧报告显示为过期，但不会被覆盖。
- LibTV 与本地管线产物使用同一人工量表，并保留分辨率、时长、帧率、音轨、来源和已知费用。

这套系统是质量决策的证据台，不是替代导演判断的“AI 总分器”。完整规则见
[评测系统](evaluation-system.md)。

## 声音选角与角色声音圣经

配音必须像演员选角一样发生在批量合成之前。每个主要角色至少试听 3 个合法可用的候选音色，
审核通过后锁定声音身份。未经授权不得克隆、模仿或暗示特定真人声音。

建议为角色资产增加 `voiceBible`：

```yaml
voiceBible:
  perceivedAge: "22–26"
  pitchRange: "mid-low"
  texture: "clean, slight breath"
  paceWpm: [190, 225]
  accent: "neutral Mandarin"
  defaultEnergy: 0.42
  sentenceEnding: "收住，不上扬"
  defensePattern: "越生气越慢、音量越低"
  relationshipModes:
    superior: "短句、谨慎、减少停顿"
    friend: "语速稍快，允许抢话"
    intimate: "音量降低，停顿变长"
  forbidden:
    - "播音腔"
    - "鸡汤式重音"
    - "每句相同的呼吸模板"
```

锁定后应保持音色、年龄感、口音和基础发声位置稳定。人物成长主要通过语速、停顿、重音、呼吸、
词汇和句尾变化表达，不通过随意更换声音。

## 台词表演数据

TTS 不应只接收孤立文本。每句台词至少应允许携带：

```yaml
performance:
  objective: "不让对方看出自己受伤"
  subtext: "我知道你在排挤我"
  addressedTo: "CH-02"
  emotion: "restrained_humiliation"
  energy: 0.36
  pace: "slow"
  pauseBeforeMs: 420
  pauseAfterMs: 680
  emphasis: ["原图"]
  breath: "shallow"
  overlap: false
  space: "crowded_backstage"
```

供应商支持时，以场景或对话轮次为单位合成；只能逐句合成时，也必须把前一句、场景目标、人物
关系和情绪残留传入提示。旁白、内心独白、电话声和现场对白使用不同声场，不得只换音量。

## 配音与混音 QC

自动检查至少覆盖：

- 台词缺失、重复、错序和字幕不一致
- 角色音色漂移、年龄感突变和口音突变
- 人名、品牌名、多音字和外语读音
- 机械等间距、错误重音、异常长停顿和句尾模板化
- 爆音、削波、底噪、过度降噪和响度突变
- 口型偏差超过项目允许范围
- 对话、音乐、环境声和拟音的可懂度与动态关系
- 同一空间内混响、距离和声像不一致

自动 QC 只能标记风险。主要角色首次定声、情绪转折场和最终成片仍需要人工试听批准。

## 剧本可表演性

关卡①除结构校验外，还应检查：

1. 每场有明确人物目标、阻力、策略变化和结果。
2. 关键情绪通过动作、选择或关系变化发生，而不是只由旁白宣布。
3. 台词允许演员隐藏信息；避免所有人物都直接说出真实想法。
4. 每集至少有一个可见、不可轻易撤销的事件。
5. 每集的情绪结论来自已发生的剧情，不是后贴的鸡汤文案。
6. 角色说话方式可区分，隐藏姓名后仍能判断大部分台词属于谁。

可表演性审核不禁止旁白，但必须回答：删掉旁白后，画面和行为是否仍然构成一场戏？

## 分镜与镜头语言

每张卡除画面描述外，应记录 `shotIntent`：

```yaml
shotIntent:
  function: "reaction"
  information: "她第一次意识到自己被裁出合照"
  audienceKnowsBeforeCharacter: false
  emotionalPressure: 0.74
  eyelineTarget: "phone-screen"
  screenDirection: "left-to-right"
  transitionIn: "match-action"
  transitionOut: "sound-bridge"
```

推荐的镜头功能枚举：

- `establish`：建立空间、关系或规则
- `reveal`：揭示新信息
- `reaction`：让观众读取人物变化
- `pressure`：压缩空间或增加时间压力
- `choice`：呈现行动前后的不可逆选择
- `payoff`：回收伏笔或完成情绪释放
- `transition`：跨越时间、空间或叙事段落

分镜批准前检查：

- 关键事件是否先给信息、再给反应，还是有明确理由反过来
- 轴线、人物视线、出入画方向和手中物连续性
- 景别是否服务情绪，而不是机械地全景—中景—近景循环
- 对白场是否保留倾听者和反应镜头
- 每次推、拉、摇、移是否有叙事动机
- 音桥、动作匹配和视觉匹配是否承担转场
- 快切是否来自冲突强度，慢镜头是否来自主观体验

## 人物连续性

人物一致性不能只检查脸：

| 维度 | 连续性要求 |
|---|---|
| 面部 | 身份特征、年龄、痣/疤、妆容逻辑 |
| 身体 | 身高比例、肩颈轮廓、步态 |
| 服装 | 场次、时间、经济状态和成长阶段 |
| 姿态 | 站姿、手势、目光习惯和防御动作 |
| 声音 | 音色、口音、语速、句尾和关系模式 |
| 价值观 | 面对同类选择时的原则及其变化原因 |

连续性报告应同时读取角色资产、剧本、分镜、候选图、视频 take 和最终音轨，不能由单张参考图
独立判断。

## 建议关卡

现有四个人工关卡保留，目标上扩展为：

1. **关卡① 剧本与表演意图**：剧情结构、可表演性、人物声音和每场目标。
2. **关卡⓪ 视觉与声音选角**：角色外形、服装、场景、音色和声音圣经。
3. **关卡②A 分镜与镜头语言**：信息顺序、镜头功能、轴线、反应和节奏。
4. **关卡②B 关键帧与声音样片**：圈图、主要角色试听、关键情绪场样片。
5. **关卡③ 成片**：画面连续性、表演、混音、字幕、AIGC 标识和交付 QC。

任何已批准的声音、表演意图或镜头意图被修改时，都必须像剧本/分镜修改一样精准撤销相关下游
状态，而不是重置整个项目或继续使用过期产物。

## 双层发行

同一集建议生成两套可追溯交付：

- **发现版**：60–90 秒，首屏事件明确，保留一个完整选择或转折，结尾给下一集问题；适合抖音
  推荐流。
- **连续观看版**：保留完整人物弧、呼吸、转场和片尾衔接；适合合集或红果连续观看。

平台时长、版权和 AIGC 标识规则在每次交付前重新核对，不能把文档中的建议当作永久平台政策。

## 实现优先级

### P0：消除通用 AI 味

- 声音候选试听、锁定和角色声音圣经
- 台词表演参数与旁白/对白声场分离
- `shotIntent`、场景目标和潜台词数据
- 主要角色声音漂移、错音、响度和对白可懂度 QC
- Studio 中的声音审核与镜头语言审核

### P1：形成导演连续性

- 上下文感知的场景级合成
- 环境声、拟音、音乐避让和空间混响
- 姿态、服装、视线、道具与声音联合连续性报告
- 修改声音/镜头意图后的精准下游失效

### P2：形成角色 IP 与发行闭环

- 发现版与连续观看版自动派生
- 角色账号资料、剧外口吻、静态图和运营日历包
- 分集钩子、情绪结论和评论反馈回收
- 只使用具有明确授权链的角色形象和声音资产
